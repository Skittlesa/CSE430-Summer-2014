Code from the "Applied Operating Systems Principles" by Silbershatz, Galvin and Gagne (also the authors of our class text) has been supplied in this zip file. This code is a semaphore solution to the classic Reader/Writers problem. The solution as provided does not prevent waiting Writers from starving. An algorithmic solution (in pseudo-code) has been provided in RWsemNoStarv.txt, which is also in the jar file. The problem associated with this approach is that it isn't very fair to the Writers because a Writer that arrives while another Writer is in the critical section, or while another Writer is already waiting, has to wait to enter the critical section until after all the Readers that are waiting, or will be waiting, first execute in the critical section.

An approach that is fairer is a queuing approach in which the arriving threads wait until they get their turn. Multiple Readers are allowed to enter simultaneously. Arriving Writers are only delayed until already queued Readers and Writers have finished.

Your assignment for this project is to:

  1. Compile and execute the given Java code ("java ReaderWriterServer" will start the program in a console) to observe starvation of Writers. Provide an output file demonstrating this phenomena.
  2. Modify the critical section entry and exit code in Database.java provided Java solution using the starvation-free psudo-code algorithm (in RWsemNoStarv.txt) so that neither Writers nor Readers will starve and so that the critical section is usable by multiple readers at a time. Compile and execute this solution to observe the reported unfairness. Provide an output file demonstrating this phenomena. You must also provide the new Java source code for Database.java.
  3. Use the RWQueue class in the jar file to modify your code from Part 2 to be fair. The RWQueue class provides a method to queue a thread and two methods to dequeue either a Reader or Writer thread. As with Part 2 you are to provide source code for the new Database.java and an output file clearly demonstrating that fair access to the critical section occurs.
